## P0 修复测试报告缺陷（本周）

- [ ] **学生端收不到老师布置的作文**（测试 B-1）
  根因：`views/student.py` 学生端整个界面没有作业列表模块——老师发布进了 `assignments` 表，但学生端从不查询该表。
  修复：在"开始写作文"页恢复"老师布置的作文题"区块，按学生年级+班级查询，**展示 prompt（布置说明）列**（同时修掉 A 版遗留的"学生看不到作文要求"问题）。

- [ ] **同一篇作文第二次改写出现错误界面**（测试 B-2）
  根因：`views/student.py` `_process_rewrite` 中 `max_ver = versions.iloc[0]["max_ver"]`，当所选作文没有版本记录时（改写产生的新 submission 从不写入 version 1），max_ver 为 NaN，`int(NaN)` 抛异常，且异常 traceback 直接 `st.code` 展示给小学生。
  修复：`next_version = int(max_ver) + 1 if pd.notna(max_ver) else 2`；面向用户只显示友好错误提示，traceback 记日志。

- [ ] **改写指导显示原始转义 JSON**（测试 B-2 截图 `["补…"]`）
  根因：LLM 返回的 `step_rewrite` 形状未校验（可能是 list 或中文键 dict），入库用 `json.dumps` 默认 `ensure_ascii=True`；读取时按固定英文键取值失败后，except 分支把转义字符串原样打印。
  修复：入库前归一化为固定四键 dict（step1_content…step4_end，缺失用 fallback 补齐），`ensure_ascii=False`；渲染时对 dict/list/str 分别处理。

- [ ] **提交 2 次改写但系统只记 1 次**（测试 B-3）
  根因：`_process_rewrite` 把改写既存为**新 submission**、又把版本追加到**原 submission**；而"历史版本对比"只查"最新一篇 submission"——最新那篇（改写件）没有任何版本记录，永远提示"只有一个版本"。
  修复：改写不再创建新 submission，只追加 `essay_versions` 并更新原 submission 的最新分数与点评；版本对比页允许选择任意一篇作文（不只最新一篇）。

- [ ] **成长档案显示作文题目**（测试 B-4）：`growth_records` 无题目字段，加 `topic` 列（或联表 submissions），列表按"题目+日期+分数"展示。

- [ ] A 版遗留 UI 项：登录栏显示学生年级/班级；教师端"布置作文题"页下方列出已布置作业历史。

## P0.5 安全整改（公开上线前必须，均为已确认漏洞）

- [ ] 移除 `main.py` 侧边栏公示的 5 个默认账号（teacher1/123456 等）；演示账号改为环境变量开关 + 环境变量密码（参考本地 v5 的 `ESSAY_APP_SEED_DEMO_USERS` 做法）。
- [ ] `config.py` 默认 `OPENAI_BASE_URL` 指向第三方代理 `4.0.wokaai.com`（学生作文会发往非官方端点），改为官方地址或强制显式配置。
- [ ] 注册权限收紧：`main.py`/`services/auth.py` 目前任何人可自助注册为 teacher；家长注册时 `parent_of` 可直接绑定任意学生（可越权看任意孩子作文）。teacher 改为邀请码/管理员创建；家长绑定需学生端或教师确认。
- [ ] `services/auth.py` bcrypt 缺失时的固定盐 SHA-256 回退改为 PBKDF2 随机盐 + `hmac.compare_digest`（v5 已有现成实现可移植）。
- [ ] 密码最少 8 位、用户名格式校验（3-32 位字母数字下划线）、登录失败限流（防爆破）。
- [ ] `services/llm.py` 加 `timeout=30` 和按用户限流（v5 的 `FixedWindowRateLimiter` 可移植）；LLM 返回 JSON 做 schema 归一化，防缺字段崩溃。
- [ ] 上传限制：txt ≤64KB；图片 ≤5MB、≤400 万像素，送模型前压缩到 1024 边长（v5 的 `uploads.py` 可移植）。

## P2 上线服务（正式部署）

- [ ] **SQLite → Supabase Postgres**（最高优先：Streamlit Cloud 容器文件系统是临时的，现在每次重启/重新部署用户数据全部丢失，收费服务绝不能建在这上面）。抽一层 repository 接口，本地开发仍可用 SQLite。
- [ ] 部署检查：应用设为公开访问（当前部署跳登录墙，家长/学生打不开）；`OPENAI_API_KEY` 走 Streamlit secrets；国内访问 streamlit.app 不稳定，正式收费后迁移国内云（阿里云/腾讯云 + Docker + 自有域名 + HTTPS）。
- [ ] 运维：每日数据库自动备份；记录 AI 调用量/失败率/每用户消耗（成本控制与限流依据）；错误日志收集。

## P3 免费/付费分层 + 扫码支付

**定价**：高级会员 **26 元/月**、**288 元/年**（折合 24 元/月）。

**免费（演示）层**：注册登录、写作文 + 规则版点评（本地评分引擎）、作文模板与推荐题目、成长档案基础表、AI 智能点评每天限 3 次试用。

**高级层**：不限次 AI 智能点评、看图作文 AI 观察提示、继续改写 + 历史版本对比、AI 成长分析报告、家长多孩绑定、拍照 OCR 作文识别（P4 上线后）。

实现步骤：
- [ ] 数据模型：`subscriptions` 表（username, plan, started_at, expires_at, status）；`orders` 表（order_no, username, plan, amount, status: pending/paid/expired, paid_at, note）。
- [ ] 功能门控：统一 `require_premium(user)` 检查（含到期判断），免费用户在高级功能处看到升级引导页；到期自动降级，到期前 7 天站内提醒。
- [ ] **支付阶段 1——扫码 + 人工核销**（最快可上线）：
  - 升级页生成订单号，展示微信/支付宝收款二维码（26 或 288 固定金额），提示用户支付备注订单号；
  - 管理员端加"订单核销"页：核对到账后一键激活订阅；
  - 备选：管理员批量生成一次性激活码线下售卖，用户输码激活。
- [ ] **支付阶段 2——自动到账**：办理个体户/企业资质后接入微信支付商户号或支付宝当面付，支付回调自动激活订阅（个人收款码长期经营性收款有合规风险，起量后必须切换）。
- [ ] 转化设计：新注册赠 7 天高级体验；免费点评结果页展示"升级解锁完整 AI 点评"对比样例。

**合规注意（面向未成年人收费）**：
- [ ] 支付页明示"请在家长指导下购买"，引导由家长账号完成支付；写明退款政策。
- [ ] 自有域名国内运营需 ICP 备案；收集个人信息遵守《个人信息保护法》与未成年人保护相关规定：上线隐私政策、信息最小化收集、家长同意机制。

## P2.5 英语写作模块（K12 小学段，v6 新功能）

- [ ] **分年级英语写作体系（三—六年级）**，与语文侧同一账号、同一成长档案：
  - 三年级：看图说话/单句描述（Picture Description、My Family、My Pet），目标 30–50 词；
  - 四年级：My Day、My School、简单日记（Diary），目标 40–60 词；
  - 五年级：My Hobby、My Best Friend、书信/邮件（Letter/Email），目标 60–80 词；
  - 六年级：小故事（Story）、My Dream、图表简单描述，目标 80–120 词。
- [ ] 英语写作模板：句型骨架（sentence starters）、常用连接词（first/next/then/finally、and/but/because）、万能开头结尾。
- [ ] 英语评分 rubric：内容切题、词汇运用、语法与拼写、结构（beginning–middle–end）、书写与标点；本地启发式评分（词数、句首大写、连接词、词汇多样性）+ LLM 精细点评。
- [ ] LLM 英语点评：英文批改 + 中文讲解的双语反馈，含 grammar_corrections（逐句语法/拼写订正）；无 API Key 时回退本地引擎。
- [ ] 界面：学生端增加"语文作文 / 英语写作"科目切换；英语提交同样计入成长档案（标记 subject）。
- [ ] 定位：英语基础写作免费试用（同 AI 每日额度），英语 AI 精批 + 语法订正为高级功能。

## P4 功能扩充（测试报告第 3 节建议，作为高级版卖点）

- [ ] **拍照上传手写作文**：OCR 识别（requirements 已预留 rapidocr）→ 错别字/标点检查 → AI 修改建议。小学生以手写为主，这是最贴合场景的付费功能。
- [ ] 固定点评维度（内容/结构/表达/创意），让 LLM 按 rubric 返回各维度分数，替换现在的关键词凑分启发式，支持个人与班级纵向统计。
- [ ] AI 学期成长分析报告（个人页面一键生成）。
- [ ] 师生互动点评（教师手动补充点评、学生回复）。
